## 4. Підхід до тестування

- **Page Object Model** — кожна сторінка інкапсульована в окремому класі для повторного використання
- **Селектори** — використовуються CSS-класи Rozetka та текстові селектори
- **Парсинг цін** — Ukrainian price format «12 999₴» → число 12999
- **Сортування через URL** — параметр `sort=cheap` / `sort=expensive` для надійності
- **Перевірка порядку** — тести сортування перевіряють відносний порядок (зростання/спадання), а не конкретні значення цін

## 5. Обмеження та ризики

| Ризик                                      | Мітигація                                                   |
| ------------------------------------------ | ----------------------------------------------------------- |
| Тести залежать від живого production сайту | Перевіряємо відносні значення, не абсолютні                 |
| Ціни та наявність товарів змінюються       | Перевіряємо лише порядок сортування та наявність елементів  |
| Можливий захист від ботів                  | Використовуємо реалістичні таймаути та user-agent браузера  |
| Попапи (cookie consent, промо)             | Обробка в `dismissPopups()` у `HomePage`                    |
| Зміна верстки/CSS-класів Rozetka           | Page Object Model дозволяє оновити селектори в одному місці |

## 6. Запуск тестів

```bash
# Запуск усіх тестів
npm test

# Запуск з відображенням браузера
npm run test:headed

# Запуск тільки в Chromium
npm run test:chromium

# Перегляд звіту
npm run report

# Запуск конкретного файлу
npx playwright test tests/search.spec.ts
npx playwright test tests/sorting.spec.ts
npx playwright test tests/product-selection.spec.ts
```
